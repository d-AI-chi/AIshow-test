# GitHubとVercelリポジトリ削除手順

## ⚠️ 注意事項

- **リポジトリを削除すると、すべてのコード、コミット履歴、デプロイ履歴が完全に削除されます**
- **この操作は取り消せません**
- **削除前に必要なデータをバックアップしてください**

## ステップ1: GitHubリポジトリを削除

### 1-1. GitHubにアクセス
1. https://github.com にアクセス
2. ログイン
3. リポジトリ一覧から「**AIshow-test**」を開く
   - URL: `https://github.com/d-AI-chi/AIshow-test`

### 1-2. リポジトリの設定を開く
1. リポジトリページの上部メニューで「**Settings**」タブをクリック
2. ページを下にスクロール

### 1-3. 危険ゾーンで削除
1. 「**Danger Zone**」（危険ゾーン）セクションまでスクロール
2. 「**Delete this repository**」（このリポジトリを削除）をクリック
3. 確認ダイアログが表示されます
4. リポジトリ名「**d-AI-chi/AIshow-test**」を入力して確認
5. 「**I understand the consequences, delete this repository**」をクリック

## ステップ2: Vercelプロジェクトを削除

### 2-1. Vercelダッシュボードにアクセス
1. https://vercel.com/dashboard にアクセス
2. ログイン

### 2-2. プロジェクトを開く
1. プロジェクト一覧から「**AIshow-test**」をクリック

### 2-3. プロジェクト設定を開く
1. プロジェクトページの上部メニューで「**Settings**」タブをクリック

### 2-4. プロジェクトを削除
1. 「**General**」セクションまでスクロール
2. ページ下部の「**Danger Zone**」セクションを確認
3. 「**Delete Project**」をクリック
4. 確認ダイアログでプロジェクト名を入力
5. 「**Delete**」をクリック

## ステップ3: ローカルのGit設定を削除（オプション）

GitHubリポジトリを削除した後、ローカルのGit設定も削除できます：

```bash
cd "/Users/daichishimizu/Documents/01. Private/AIshow-test"

# リモート接続を削除
git remote remove origin

# .gitフォルダを削除（Git履歴を完全に削除）
# 注意: この操作は取り消せません
rm -rf .git
```

## 削除後の影響

### GitHubリポジトリ削除後
- ✅ ローカルのコードは残ります
- ❌ GitHub上のコードは完全に削除されます
- ❌ コミット履歴、Issue、Pull Requestがすべて削除されます

### Vercelプロジェクト削除後
- ❌ デプロイされたサイトが削除されます
- ❌ デプロイ履歴がすべて削除されます
- ❌ 環境変数の設定が削除されます

## 削除後の対応

### 新しいリポジトリを作成する場合
削除後、必要に応じて新しいリポジトリを作成できます。

### ローカルのコードを保持する場合
ローカルのコードは削除されません。必要に応じて新しいリポジトリに接続できます。

## 確認チェックリスト

削除前に確認：
- [ ] 必要なコードをバックアップしたか
- [ ] 環境変数の値をメモしたか（Vercelの場合）
- [ ] 本当に削除してよいか確認したか

削除後：
- [ ] GitHubリポジトリが削除されたか確認
- [ ] Vercelプロジェクトが削除されたか確認
- [ ] ローカルのコードが残っているか確認
