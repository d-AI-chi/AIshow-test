# Vercelドメインとデプロイの説明

## 2つのリンクの違い

### 1つ目のリンク（`git-main`）
- **種類**: 本番環境（Production）
- **更新タイミング**: `main` ブランチにプッシュしたときに自動更新
- **用途**: 実際に使用する本番サイトのURL
- **特徴**: 常に最新の `main` ブランチの内容が表示される

### 2つ目のリンク（`pwp4oqbhK`）
- **種類**: プレビューデプロイ（Preview）
- **更新タイミング**: 特定のコミットやブランチのデプロイ
- **用途**: テストや確認用の一時的なURL
- **特徴**: 特定のコミットの内容が固定される

## 1つ目のリンクが開けない理由

### 考えられる原因

1. **デプロイが失敗している**
   - ビルドエラーが発生している可能性
   - 確認方法: 「Deployments」タブで最新のデプロイの状態を確認

2. **デプロイが削除された**
   - 手動で削除された、または期限切れの可能性
   - 確認方法: 「Deployments」タブでデプロイ履歴を確認

3. **デプロイがまだ完了していない**
   - ビルド中の可能性
   - 確認方法: 「Deployments」タブで「Building」状態を確認

4. **ドメインの設定に問題がある**
   - まれですが、ドメインの設定に問題がある可能性

## 確認と対処方法

### ステップ1: デプロイ状態を確認

1. Vercelダッシュボードでプロジェクトを開く
2. 「**Deployments**」タブをクリック
3. 最新のデプロイを確認：
   - 🟢 **Ready**: デプロイ完了（正常）
   - 🟡 **Building**: ビルド中（待つ）
   - 🔴 **Error**: エラー発生（確認が必要）
   - ⚪ **Canceled**: キャンセルされた

### ステップ2: 本番環境のデプロイを確認

1. 「Deployments」タブで、`main` ブランチのデプロイを探す
2. 最新のデプロイの状態を確認
3. エラーがある場合は、エラーログを確認

### ステップ3: 再デプロイする

#### 方法1: コードをプッシュして自動デプロイ
```bash
# 何か小さな変更を加える（例: コメント追加）
git add .
git commit -m "デプロイテスト"
git push origin main
```

#### 方法2: 手動で再デプロイ
1. 「Deployments」タブを開く
2. 最新のデプロイの「...」メニューをクリック
3. 「Redeploy」を選択

## 使い分けの推奨

### 本番環境（1つ目のリンク）を使用する場合
- ✅ 実際にユーザーに公開するサイト
- ✅ 参加者に共有するリンク
- ✅ 管理者ページへのアクセス
- ✅ 最終的な動作確認

### プレビュー環境（2つ目のリンク）を使用する場合
- ✅ 開発中の機能をテスト
- ✅ 他のブランチの変更を確認
- ✅ デプロイ前の最終確認
- ✅ 一時的な共有

## 推奨される運用方法

1. **本番環境のURLを固定して使用**
   - 1つ目のリンク（`git-main`）を本番URLとして使用
   - このURLを参加者や管理者に共有

2. **プレビュー環境は開発用**
   - 2つ目のリンクは開発・テスト用として使用
   - 本番環境に影響を与えずにテストできる

3. **定期的にデプロイ状態を確認**
   - 「Deployments」タブで定期的に確認
   - エラーが発生した場合はすぐに対処

## トラブルシューティング

### リンクが開けない場合のチェックリスト

- [ ] 「Deployments」タブで最新のデプロイの状態を確認
- [ ] エラーログを確認（エラーがある場合）
- [ ] ローカルで `npm run build` が成功するか確認
- [ ] 環境変数が正しく設定されているか確認
- [ ] 最新のコミットが正しくプッシュされているか確認

### よくあるエラーと対処法

1. **ビルドエラー**
   - エラーログを確認
   - ローカルでビルドして問題を特定

2. **環境変数のエラー**
   - 「Settings」→「Environment Variables」で確認
   - 必要な環境変数が設定されているか確認

3. **タイムアウト**
   - ビルド時間が長すぎる可能性
   - 不要なファイルを削除する
